关系过程场：一种从关系生成“人”的模拟架构

设计理念、系统架构与运行逻辑

RPF / Relational Process Field

摘要

本项目试图建立一种不同于传统“人物驱动”叙事模拟的生成模型。它不从完整的个人、固定的人格或预设的关系变量出发，而从场、过程、绑定、场景、观察、承认、稳定化与不可逆历史出发，模拟关系如何逐步稳定成可被识别的“人”、身份与命运。RPF 的核心主张是：人不是最底层的因果单元，而是关系过程在身体、制度、物质、记忆和承认结构中反复沉积后形成的高层界面。

1. 问题意识 2. 本体论反转 3. 架构分层 4. 运行逻辑 5. 数据与因果边界 6. 叙事渲染边界 7. 项目价值

1. 问题意识：不再把“人”当作起点

许多叙事模拟系统默认从人物开始：人物拥有目标、性格、情绪和关系数值，然后在环境中做出行动。但这种建模方式容易把人际关系简化为两个稳定主体之间的变量交换，也容易让“信任下降”“怨恨上升”“性格内向”等表层描述反过来成为因果原因。RPF 选择反向提问：如果一个人的可见性格、关系位置和命运感并不是先验给定的，而是在持续的关系过程中被生产出来的，那么模拟系统应该从哪里开始？

项目的回答是从关系过程场开始。所谓场，不是背景板；它包含物质依赖、制度分类、空间安排、时间压力、社会观众、记忆残留与可行动空间。所谓过程，也不是某个人的心理状态；它是反复出现的差异、误认、承认索取、回避、修复和不可逆后果。人物只是这些过程在某个时刻被投影成的可读界面。

2. 本体论反转：从“人形成关系”到“关系稳定成人”

RPF 的最重要反转可以概括为：不要模拟人如何形成关系，而要模拟关系过程如何稳定成人。系统内部的基础对象不是 Person，而是 FieldState、ProcessState、CoPresenceBinding、RelationalProcessPattern、SceneState、Event 和 IrreversibilityRegister。PersonView 与 RelationshipView 仍然存在，但它们是投影，是为了检查、阅读和叙事而生成的高层摘要。

这意味着“冷漠的人”“控制欲强的人”“亏欠的关系”“低信任关系”都不应该直接驱动行为。它们只有在被说出、被相信、被制度记录、被第三方传播，或被纳入自我叙事时，才会通过 OperativeClassificationEvent 进入因果场。标签不是魔法变量；标签必须获得社会或叙事上的承接，才会改变未来的言说风险、行动可得性、记忆解释和场景压力。

微信号 → 生成参数 → 关系过程模式 → 关系聚合 → 人物/关系视图 → 叙事标签 → 操作性分类 → 场与互动约束 → 新的微信号

3. 架构分层：九个相互约束的层

RPF 不是平面的变量系统，而是递归分层架构。物质再生产层回答谁承担劳动、谁拥有退出能力、谁支付修复成本；场位置层回答谁能定义局面、谁的痛苦被承认、谁的欲望合法；身体-惯习层回答什么话在说出口之前就已经变得危险；相关性层决定什么细节突然成为证据；共在绑定层解释为什么双方不能简单从彼此世界中消失。

在更可见的层面，互动仪式层决定一个场景里什么可以说、什么只能伪装成玩笑、沉默或实际帮助；沟通观察层处理二阶观察，即我如何理解你如何看我；承认与误认层构成戏剧引擎；稳定化与不可逆层则把重复的过程沉积为身份、关系阶段和命运式轨迹。

物质再生产 场位置 身体-惯习 相关性实现 共在绑定 互动仪式 沟通观察 承认/误认 稳定化/不可逆

4. 运行逻辑：事件驱动的关系演化

模拟以 tick 推进，但 tick 不是一句话、一个回合或固定时间步，而是一次可回放的因果事务。每个 tick 根据当前场压力、绑定强度、RPP 激活、记忆压力和可行动空间，选择潜伏、微交互或场景三种形态之一。潜伏 tick 处理等待、回避、未回复、疲劳和压力累积；微交互 tick 处理眼神、短促回答、读而不回、物件摆放等小信号；场景 tick 则处理争执、道歉、协商、公开表演、暴露或离开。

一个完整场景不会直接问“这个人想做什么”，而会问“在这个位置、这个场、这个观众结构和这个历史压力下，哪些行动作为行动变得可用”。行动还要经过表达选择，变成语气、停顿、姿态、沉默或公开面具。随后系统执行观察、承认评估、修复或回避、稳定化、不可逆记录、聚合与投影。所有关键变化都写入事件流，使 replay 能从初始状态与事件历史重建因果状态。

5. 数据模型与因果边界

项目把数据分为三类：可写因果状态、追加式事实、派生视图。FieldState、ProcessState、ActiveRPP、SceneState、IrreversibilityRegister 和 OperativeClassification 可以参与状态更新；Event、EpisodeRecord 和 Snapshot 作为历史事实追加；PersonView、RelationshipView 与 RelationalAggregateView 只能从下层事件和状态重新计算。

这条边界保护了项目的研究目标：信任、亲密、依赖、怨恨、权力不对称和人格都不是原始变量。系统可以展示“信任降低”，但真正发生变化的是风险悬置范围、检查频率、歧义容忍度、受伤记忆、修复债和言说抑制。高层词汇负责让人读懂系统，不负责偷走系统的因果权。

6. LLM 边界：只渲染事实，不发明因果

RPF 可以使用大语言模型生成故事文本，但 LLM 的职责被严格限制在渲染层。它可以把当前场景、人物视图、关系视图、活跃 RPP、不可逆记录和故事帧写成中文叙事；它不能决定谁道歉、谁误认、哪个标签成为因果事实，也不能绕过事件 schema 修改模拟状态。

因此，LLM 在这里不是“剧情发动机”，而是“表层叙事显影器”。它负责把深层结构翻译成可读的场景质感、对白候选、沉默、动作和有限视角，而模拟器仍然保留因果权威。这使项目既能获得文学化表达，又能维持可验证、可回放、可诊断的结构完整性。

7. 项目价值：让关系成为真正的生成对象

RPF 的目标不是生成一段看似合理的故事，而是建立一种可以观察人际过程如何变成身份、关系和命运的实验环境。它适合研究长期关系中的误认、照护负担、沉默解释、公开与私下分裂、阶层差异、债务锁定、修复失败和不可逆转折。它也适合创作，因为它不依赖预设剧情，而从约束、承认、记忆和场景压力中让故事浮现。

最终，这个项目试图保留人的复杂性：人既不是自由漂浮的意志，也不是固定性格的容器；人是在关系场中被迫行动、被他人识别、被历史压缩、又仍然可能改变的过程性存在。RPF 要模拟的，正是这种介于约束与生成之间的关系生命。
